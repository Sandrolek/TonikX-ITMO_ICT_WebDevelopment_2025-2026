# Отчет по лабораторной работе 2 - Django веб-приложение

## Цель работы

Разработка полнофункционального веб-приложения для туристического агентства с использованием фреймворка Django. Изучение принципов работы с моделями данных, представлениями, формами и шаблонами в Django.

## Описание проекта

Веб-приложение для туристического агентства, позволяющее пользователям просматривать туры, делать резервирования и оставлять отзывы. Администраторы могут управлять турами и подтверждать резервирования.

## Основная функциональность

### Для пользователей:
- Регистрация и авторизация
- Просмотр списка туров с поиском и пагинацией
- Детальная информация о турах
- Создание резервирований
- Редактирование и удаление своих резервирований
- Добавление отзывов с рейтингом (1-10)

### Для администраторов:
- Управление турами через админ-панель Django
- Просмотр всех резервирований
- Подтверждение и отмена резервирований
- Просмотр статистики по проданным турам по странам

## Используемые технологии

- **Django 5.2.7** - веб-фреймворк
- **PostgreSQL** - база данных
- **Bootstrap 5** - CSS фреймворк для UI
- **Font Awesome** - иконки
- **Docker Compose** - для запуска PostgreSQL

## Архитектура проекта

Проект организован по стандартной структуре Django:

```
Lr2/
├── Lr2/              # Основные настройки проекта
│   ├── settings.py   # Конфигурация приложения
│   ├── urls.py       # Главный URL роутер
│   └── ...
├── tourism/          # Основное приложение
│   ├── models.py     # Модели данных
│   ├── views.py      # Представления (контроллеры)
│   ├── urls.py       # URL маршруты приложения
│   ├── forms.py      # Формы
│   ├── admin.py      # Настройки админ-панели
│   └── ...
├── templates/        # HTML шаблоны
│   └── tourism/
├── static/           # Статические файлы (CSS, JS, изображения)
└── manage.py         # Скрипт управления Django
```

## Ключевые особенности реализации

1. **Модели данных**: Tour, Reservation, Review с связями ForeignKey
2. **Пагинация**: Использование Django Paginator для списков
3. **Поиск**: Фильтрация по названию, стране и агентству
4. **Валидация**: Проверка доступности мест при бронировании
5. **Авторизация**: Использование декораторов Django для защиты маршрутов
6. **Статусы резервирований**: pending, confirmed, cancelled

## Результаты

Успешно реализовано полнофункциональное Django веб-приложение с регистрацией пользователей, системой резервирований, отзывами и административной панелью. Изучены основные концепции Django: ORM, представления на основе функций, формы, шаблоны и система аутентификации.
