# Туристическое агентство - Django приложение

## Описание проекта

Веб-приложение для туристического агентства, позволяющее пользователям просматривать туры, делать резервирования и оставлять отзывы.

## Функциональность

- **Регистрация и авторизация пользователей**
- **Просмотр списка туров** с поиском и пагинацией
- **Детальная информация о турах**
- **Резервирование туров** с возможностью редактирования и удаления
- **Система отзывов** с рейтингом (1-10)
- **Административная панель** для управления турами и подтверждения резервирований
- **Отчет по проданным турам** по странам

## Технологии

- Django 5.2.7
- PostgreSQL
- Bootstrap 5
- Font Awesome

## Установка и настройка

### Быстрый старт с Docker (Рекомендуется)

#### 1. Запуск PostgreSQL через Docker Compose

```bash
docker-compose up -d
```

Это создаст и запустит контейнер PostgreSQL с базой данных `tourism_db`.

#### 2. Установка зависимостей

```bash
pip install -r requirements.txt
```

#### 3. Применение миграций

```bash
python manage.py makemigrations
python manage.py migrate
```

#### 4. Создание суперпользователя

```bash
python manage.py createsuperuser
```

#### 5. Запуск Django сервера

```bash
python manage.py runserver
```

База данных уже настроена и готова к использованию!

### Установка без Docker

#### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

#### 2. Настройка базы данных PostgreSQL

Создайте базу данных PostgreSQL:

```sql
CREATE DATABASE tourism_db;
CREATE USER postgres WITH PASSWORD 'postgres';
GRANT ALL PRIVILEGES ON DATABASE tourism_db TO postgres;
```

#### 3. Настройка переменных окружения (опционально)

Вы можете создать файл `.env` в корне проекта с настройками базы данных:

```
DB_NAME=tourism_db
DB_USER=postgres
DB_PASSWORD=postgres
DB_HOST=localhost
DB_PORT=5432
```

По умолчанию настройки в `Lr2/settings.py` используют значения выше.

#### 4. Миграции

```bash
python manage.py makemigrations
python manage.py migrate
```

#### 5. Создание суперпользователя

```bash
python manage.py createsuperuser
```

#### 6. Запуск сервера

```bash
python manage.py runserver
```

### Остановка Docker контейнера

```bash
docker-compose down
```

Для полного удаления данных (включая базу данных):

```bash
docker-compose down -v
```

## Модели данных

### Tour (Тур)
- Название тура
- Турагенство
- Описание
- Период проведения
- Условия оплаты
- Страна
- Цена
- Максимальное количество участников

### Reservation (Резервирование)
- Связь с туром и пользователем
- Количество участников
- Статус (ожидает/подтверждено/отменено)

### Review (Отзыв)
- Связь с туром и пользователем
- Рейтинг (1-10)
- Комментарий
- Дата тура

## Использование

1. **Администратор** может:
   - Добавлять новые туры через админ панель
   - Подтверждать резервирования
   - Управлять пользователями

2. **Пользователи** могут:
   - Просматривать туры
   - Делать резервирования
   - Редактировать/удалять свои резервирования
   - Оставлять отзывы

## Особенности

- **Responsive дизайн** с Bootstrap 5
- **Пагинация** для списков туров и отзывов
- **Поиск** по названию, стране и агентству
- **Валидация** количества участников при бронировании
- **Система статусов** для резервирований
- **Отчетность** по проданным турам
